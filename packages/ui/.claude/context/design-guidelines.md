# Design Guidelines — Unlighthouse UI

> Generated by `nuxt-frontend-design`. This file captures the *intent* behind design decisions so future work stays cohesive across conversations, contributors, and skills.

## Design Principle

**We prioritize data legibility over visual theatrics.** Glass and ambient depth set atmosphere, but every score, metric, and audit row must read instantly. When a design choice trades clarity for polish, clarity wins.

## Aesthetic Direction

- **Theme**: frost (adapted) — frost glassmorphism toned down for dashboard density. Glass on cards and overlays only; data surfaces stay flat and high-contrast.
- **Mode**: dark only (`colorMode.preference: 'dark'`, `ssr: false`)
- **Vibe**: clinical glass dashboard, ambient cool, observatory at night
- **Influences**: Linear's data density, Vercel Analytics' dark calm, Lighthouse's status semantics, frost's translucent layering

## Color Decisions

| Role | Value | Why |
|------|-------|-----|
| Primary | `cyan` | Cool, technical, distinct from product data colors (LCP indigo, INP sky, CLS purple). Reads as "system action" not "data signal". |
| Secondary | `blue` | Pairs with cyan in frost mesh; used sparingly for secondary CTAs. |
| Neutral | `slate` | Cool-tinted neutral matches the frost atmosphere; warm neutrals would fight the cyan/blue mesh. |
| Success | `green` | Lighthouse score >=90 convention; preserved from prior code. |
| Warning | `amber` | Lighthouse score 50–89 convention. |
| Error | `red` | Lighthouse score <50, critical/serious severity. |
| Info | `sky` | Distinguishes informational badges from primary actions. |

- **Neutral tinting**: using Tailwind `slate` palette (cool, blue-tinted). Custom `--ui-bg: #08090b` is near-black with a slight blue cast — never pure `#000`.
- **60-30-10 split**: 60% near-black bg + slate text scale, 30% glass surfaces and borders (white at 4–14% opacity), 10% cyan primary on CTAs and active states. Status colors (green/amber/red) are reserved for score and severity semantics, not decoration.

### Contrast & Accessibility

- **Body text contrast**: `--ui-text` at `rgba(255,255,255,0.92)` over `#08090b` ≈ 18.5:1 — exceeds AAA.
- **Muted text**: `rgba(255,255,255,0.62)` ≈ 9:1 — exceeds AA Large; use only for secondary metadata, never for primary metrics.
- **Dark mode adjustments**: no pure black backgrounds; semitransparent glass surfaces over dark base; status colors (red/amber/green) used at 400 weight in text contexts to avoid retinal burn.
- **Known risks**: glass surfaces over the mesh background can drop contrast — verify any text on `.glass` panels stays >= 4.5:1. Audit any new badge variants against muted bg.

## Typography

| Role | Font | Why |
|------|------|-----|
| Body (`--font-sans`) | Satoshi | Geometric, modern; reads as engineered/measured (vs Inter's ubiquity). Loaded via Nuxt Fonts. |
| Display (`--font-display`) | not used | Dashboard UI — display fonts add noise without serving comprehension. Headings use `--font-sans` at heavier weights. |
| Mono (`--font-mono`) | JetBrains Mono | Used for URLs, response codes, route paths, raw audit output. Pairs visually with Satoshi. |

- **Type system**: fixed rem scale — this is application UI, not marketing. Avoid fluid `clamp()` for table/dashboard text.
- **OpenType features**: `font-variant-numeric: tabular-nums` on every metric, score, and table number column. Use the `.nums-tabular` utility.

## Icons

- **Collection**: `heroicons` (primary), `lucide` (fallback for icons heroicons lacks)
- **Why**: heroicons' 1.5px outline style glows cleanly on dark glass and matches the restrained aesthetic; lucide covers gaps (e.g., `chevrons-left-right`, charting glyphs)
- **Color rule**: monochrome only via `text-current`. Color is reserved for semantic states (success/warning/error icons in audit rows) and active nav items (primary).

## Component Rules

- **Cards**: glass by default (`backdrop-blur-xl bg-white/[0.04] border border-white/[0.08]`). Never use hard solid backgrounds — they look pasted onto the mesh.
- **Buttons**: solid variant for primary actions, outline for secondary. Ghost only inside dense toolbars where solid would crowd. Primary solid emits a cyan glow on hover (frost signature).
- **Radius**: `--ui-radius: 0.625rem` (10px) globally. Tighter than frost's default 12px because dashboards have more nested rectangles. `rounded-full` only for avatars and status dots.
- **Inputs**: glass backgrounds with white/10 border; focus border shifts to `primary/50`. No box-shadows on focus — the border color change is the affordance.
- **Status badges**: always soft variant. Use semantic color props (`color="success"`) over hardcoded Tailwind colors.
- **Numbers**: every metric, score, latency, byte count — tabular figures. Apply `.nums-tabular` or `font-variant-numeric: tabular-nums`.
- **Tables**: header row uses `bg-elevated`; rows separated by `border-b border-default`. No zebra striping — it competes with severity row highlighting.
- **Modals**: heaviest blur (`backdrop-blur-2xl`) with semitransparent overlay. Always include explicit close affordance.
- **Error messages**: 3-part where possible (what / why / fix). Never blame the user.

## Spatial & Motion

- **Spacing system**: 4pt base grid: 1 (4px), 2 (8px), 3 (12px), 4 (16px), 6 (24px), 8 (32px), 12 (48px), 16 (64px). Avoid `5`, `7`, `9`, `10`, `11` — they break rhythm.
- **Spacing philosophy**: data-dense dashboards use compact spacing (`p-3` to `p-4` for cards, `gap-2` to `gap-3` in lists). Reserve generous spacing (`py-12`+) for landing/onboarding pages only.
- **Transition speed**: 150–250ms for micro-interactions (hovers, focus); 300ms for state changes (modals, drawers). No animations longer than 400ms in dashboard surfaces.
- **Animation style**: opacity + 4px translateY for entrance; spring scale on hover for interactive cards (`scale-[1.01]`). Stagger only on initial page load, never on data refresh.
- **Reduced motion**: `prefers-reduced-motion` collapses all entrance animations to instant opacity. Border-glow hovers preserved as static color shifts. Score gauge animation disabled.

## Responsive Strategy

- **Approach**: desktop-first dashboard. Primary breakpoints: `md` (768px) and `lg` (1024px). Mobile (375px) is functional but not optimized — collapse sidebars to drawers, stack columns.
- **Input method**: pointer-primary. Hover effects gated by `@media (hover: hover)` to avoid sticky states on touch.
- **Navigation adaptation**: sidebar collapses to icon rail at `md`, becomes overlay drawer below `md`. Top nav stays horizontal at all sizes.

## Voice & Tone

- **Button labels**: verb + object: `Run scan`, `Export results`, `View report`. Never `OK`, `Submit`, or bare verbs.
- **Empty states**: 3-part structure — acknowledge ("No scans yet") + explain value ("Audits will appear here once a scan completes") + provide action (CTA button).
- **Error style**: 3-part where possible: `Scan failed — connection refused on port 3000. Check the server is running and retry.`
- **Score copy**: never say "good"/"bad" alone. Pair with the number: `92 — passing` not `Good`.
- **Severity labels**: critical / serious / moderate / minor / info — match audit conventions, don't soften.

## Avoid

- **Hardcoded Tailwind colors** (`bg-red-500`, `text-amber-400`, `border-gray-500`) — use semantic props (`color="error"`) or CSS vars (`bg-error/10`, `text-error`). Exception: social-platform preview cards in `pages/results/[scanId]/seo.vue` intentionally mimic Google / X / Facebook / LinkedIn / Slack / Discord brand colors and surfaces. Never repurpose those tokens elsewhere.
- **Inter, Roboto, Open Sans, Lato, Montserrat, system-ui** — banned. Use Satoshi for sans, JetBrains Mono for mono.
- **Pure black** (`#000`) and **pure white** (`#fff`) for surfaces — always use the tinted scale.
- **Solid card backgrounds** (`bg-slate-900`, `bg-gray-800`) — breaks the glass language. Use `.glass` or `<UCard>`.
- **Drop shadows on dark surfaces** — use border-opacity contrast or the colored cyan glow on primary actions.
- **Colorful icons** outside semantic states — looks like a toy dashboard. Monochrome via `text-current`.
- **Zebra striping in tables** — competes with severity row highlighting.
- **Bouncy / elastic easing** — use ease-out or springs with damping. This is an audit tool, not a toy.
- **Nested cards** — flatten with spacing and dividers instead. Card-in-card halves the glass effect and adds visual noise.
- **Skeleton loaders that pulse aggressively** — use a slow shimmer (1.5s+) so they don't compete with gauge animations.
- **CWV metric colors as decoration** — `--color-mesh-purple` etc are reserved for the LCP/INP/CLS chart series. Don't repurpose for unrelated UI.

## Custom Utilities

| Class/Token | What it does | When to use |
|-------------|-------------|-------------|
| `.glass` | `backdrop-blur(24px)` + `bg-white/[0.04]` + `border-white/[0.08]` | Default panel surface when not using `<UCard>` |
| `.glass-elevated` | Heavier blur, brighter bg, drop shadow | Modals, popovers, sticky toolbars over content |
| `.bg-mesh` | Radial cyan/blue/purple gradient | Page-level background on landing, onboarding, results overview |
| `.text-gradient-score` | Cyan → blue gradient text | Hero score readouts only — overuse cheapens it |
| `.nums-tabular` | `font-variant-numeric: tabular-nums` | All numeric readouts (scores, metrics, table cells) |
| `--color-glass-bg` / `--color-glass-border` | Glass surface tokens | Theme-specific; no Tailwind equivalent |
| `--color-mesh-cyan` / `-blue` / `-purple` | Mesh gradient color stops | Background mesh only |

## Design Decisions

- **Severity mapping collapses to 3 buckets**: `critical` + `serious` → `error`, `moderate` → `warning`, `minor` + `info` → `info`. Rationale: semantic palette only exposes 3 status colors and the Nuxt UI `info` token is already reserved for neutral/meta. Stronger signal per bucket beats a 5-shade spread. Applied in `SeverityBadge.vue`, `accessibility.vue` severity meta, best-practices route counts.
- **Web-vital "needs improvement" uses `warning` (amber)**: aligned with Lighthouse and CrUX convention. Previously cyan (primary) — rejected because cyan reads as system action, not caution. Applied in `performance.vue` getVitalColor/Bg, `summaryStats`, and distribution bars in `CruxMetricCard.vue`.
- **Logo tile gradient is `from-primary to-secondary`** (cyan → blue): replaces the legacy `from-primary-500 to-orange-600`. Stays inside the approved frost palette while preserving tile distinctiveness vs. flat solid buttons. Applied in `DashboardShell.vue`, `onboarding.vue`, `scan.vue`, and index.vue progress bars.
- **Tooltip surface uses `.glass-elevated`**: replaces the old `bg-gray-900/99` solid treatment for consistency with modal overlay treatment from the guidelines.
- **Dark-only light-mode fallbacks dropped**: `ssr: false` + `colorMode.preference: 'dark'` means light-mode pairs (`bg-white dark:bg-gray-900`, `bg-gray-100 dark:bg-gray-800`, `from-sky-50/50 to-sky-300/50`) are dead code. Stripped from `ModalThumbnails.vue`, `LighthouseThreeD.vue`, `error.vue`.
- **Social preview cards keep raw brand colors**: SEO `pages/results/[scanId]/seo.vue` lines 236-241 (brand icons) and 499-570 (Google/X/Facebook/LinkedIn/Slack/Discord preview cards) intentionally use `text-blue-500`, `bg-gray-100`, `bg-neutral-900` etc. to mimic real platform UI. These are documentation surfaces, not design-system surfaces — do not "fix" them.
